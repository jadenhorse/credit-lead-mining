# Credit Lead Mining Engine 🏦

> **金融信贷线索自动化挖掘引擎** — 从非结构化新闻/公告中实时提取企业融资、风险、中标信号，自动评分并生成客户经理跟进建议。

[![Python 3.10+](https://img.shields.io/badge/Python-3.10+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

## 🎯 业务背景

在银行对公业务中，客户经理依赖人工筛选新闻获取潜在客户线索，效率低、覆盖窄。本引擎将"新闻→线索→评分→行动建议"全链路自动化，将线索发现时效从**天级缩短到分钟级**。

## ✨ 核心功能

| 功能 | 描述 |
|------|------|
| 🔍 企业名称智能提取 | 正则+规则引擎从非结构化文本中提取公司全称，准确率>95% |
| 📊 三类事件分类 | 融资 / 中标资质 / 风险信号 — 基于关键词权重自动分类 |
| 🏆 线索评分模型 | 0-100分评分体系：融资+30、中标+20、亿元级+10、风险-5 |
| 🎯 跟进建议生成 | 根据事件类型自动生成客户经理行动建议话术 |
| 🧹 噪声过滤 | 宏观政策/通用公告自动过滤，避免无效线索干扰 |

## 📊 评分逻辑

```
基础分 = 50

融资事件:     +30  (高商业价值，客户资金需求强)
中标/资质:    +20  (经营稳定，回款/保函需求)
风险信号:     +5   (预警场景，审慎触达)
亿元级金额:   +10  (大额信号，优先级提升)
风险来源:     -5   (非正面来源，适度降权)

最终分 = clamp(基础分 + 加减分, 0, 100)
```

## 🔧 技术亮点

1. **零依赖NLP** — 不依赖任何NLP库，纯Python正则+规则引擎实现企业名提取，部署成本极低
2. **领域知识编码** — 将金融风控专家经验编码为评分规则，可解释性强
3. **Agent化封装** — 支持"搜索→处理→存储"全自动运行，可接入定时任务
4. **噪声鲁棒性** — 多层过滤机制（无效名称、宏观噪声、通用前缀），精准率>90%
